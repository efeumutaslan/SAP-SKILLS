# SAP HANA Cloud Monitoring Views Reference

## System & Memory

| View | Purpose | Key Columns |
|------|---------|-------------|
| `M_DATABASE` | Database info | DATABASE_NAME, VERSION, USAGE |
| `M_HOST_RESOURCE_UTILIZATION` | CPU/memory utilization | USED_PHYSICAL_MEMORY, FREE_PHYSICAL_MEMORY, CPU_USAGE_PCT |
| `M_SERVICE_MEMORY` | Memory per service | SERVICE_NAME, TOTAL_MEMORY_USED_SIZE, HEAP_MEMORY_USED_SIZE |
| `M_SERVICE_STATISTICS` | Service runtime stats | SERVICE_NAME, REQUESTS_PER_SEC, ACTIVE_THREAD_COUNT |
| `M_DISK_USAGE` | Disk consumption | USAGE_TYPE, USED_SIZE, TOTAL_SIZE |

## SQL Performance

| View | Purpose | Key Columns |
|------|---------|-------------|
| `M_SQL_PLAN_CACHE` | Cached SQL plans | STATEMENT_HASH, EXECUTION_COUNT, AVG_EXECUTION_TIME |
| `M_SQL_PLAN_CACHE_OVERVIEW` | Plan cache summary | PLAN_CACHE_HIT_RATIO, EVICTED_PLAN_COUNT |
| `M_EXPENSIVE_STATEMENTS` | Slow queries log | STATEMENT_STRING, DURATION_MICROSEC, RECORDS |
| `M_ACTIVE_STATEMENTS` | Currently running SQL | STATEMENT_ID, STATEMENT_STRING, DURATION_MICROSEC |
| `M_BLOCKED_TRANSACTIONS` | Lock wait info | BLOCKED_CONNECTION_ID, LOCK_OWNER_CONNECTION_ID |

## Table & Column Store

| View | Purpose | Key Columns |
|------|---------|-------------|
| `M_CS_TABLES` | Column store tables | TABLE_NAME, RECORD_COUNT, TABLE_SIZE |
| `M_CS_COLUMNS` | Column-level stats | COLUMN_NAME, MEMORY_SIZE_IN_TOTAL, LOADED |
| `M_CS_ALL_COLUMNS` | All column details | PART_ID, MEMORY_SIZE_IN_MAIN, MEMORY_SIZE_IN_DELTA |
| `M_RS_TABLES` | Row store tables | TABLE_NAME, RECORD_COUNT, TABLE_SIZE |
| `M_TABLE_PARTITIONS` | Partition info | TABLE_NAME, PART_ID, PARTITION_SPEC |

## Connections & Sessions

| View | Purpose | Key Columns |
|------|---------|-------------|
| `M_CONNECTIONS` | All connections | CONNECTION_ID, CONNECTION_STATUS, USER_NAME |
| `M_SESSIONS` | Active sessions | SESSION_ID, USER_NAME, SCHEMA_NAME |
| `M_TRANSACTIONS` | Open transactions | TRANSACTION_ID, TRANSACTION_STATUS, UPDATE_TRANSACTION_ID |
| `M_LOCK_WAITS_STATISTICS` | Lock wait history | LOCK_TYPE, TOTAL_LOCK_WAIT_COUNT, TOTAL_LOCK_WAIT_DURATION |

## Common Monitoring Queries

### Memory Check
```sql
SELECT
  ROUND(USED_PHYSICAL_MEMORY/1024/1024/1024, 2) AS used_gb,
  ROUND(FREE_PHYSICAL_MEMORY/1024/1024/1024, 2) AS free_gb,
  ROUND(USED_PHYSICAL_MEMORY * 100.0 /
    (USED_PHYSICAL_MEMORY + FREE_PHYSICAL_MEMORY), 1) AS used_pct
FROM M_HOST_RESOURCE_UTILIZATION;
```

### Top Slow Queries (last 24h)
```sql
SELECT TOP 20
  STATEMENT_HASH,
  ROUND(AVG_EXECUTION_TIME/1000, 2) AS avg_ms,
  EXECUTION_COUNT,
  ROUND(TOTAL_EXECUTION_TIME/1000000, 2) AS total_sec,
  SUBSTR(STATEMENT_STRING, 1, 100) AS query
FROM M_SQL_PLAN_CACHE
WHERE LAST_EXECUTION_TIMESTAMP > ADD_SECONDS(CURRENT_TIMESTAMP, -86400)
ORDER BY AVG_EXECUTION_TIME DESC;
```

### Table Size Report
```sql
SELECT TOP 30
  SCHEMA_NAME, TABLE_NAME,
  ROUND(TABLE_SIZE/1024/1024, 2) AS size_mb,
  RECORD_COUNT,
  ROUND(RAW_RECORD_COUNT_IN_DELTA * 100.0 /
    NULLIF(RECORD_COUNT, 0), 1) AS delta_pct
FROM M_CS_TABLES
ORDER BY TABLE_SIZE DESC;
```

### Active Long-Running Queries
```sql
SELECT
  CONNECTION_ID,
  ROUND(DURATION_MICROSEC/1000000, 2) AS duration_sec,
  STATEMENT_STATUS,
  SUBSTR(STATEMENT_STRING, 1, 150) AS query
FROM M_ACTIVE_STATEMENTS
WHERE DURATION_MICROSEC > 5000000  -- > 5 seconds
ORDER BY DURATION_MICROSEC DESC;
```
